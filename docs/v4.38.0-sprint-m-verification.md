## 🧩 功能模块详情

### 1. AI色彩分级 (`ai/color-grading.ts`)

| 功能 | 函数 | 说明 |
|------|------|------|
| 色彩分析 | `analyzeImageColors()` | 分析亮度/对比度/饱和度/色温/色调/主色调/直方图/动态范围 |
| 智能色彩匹配 | `matchColors()` | 不同镜头间色彩风格匹配，支持直方图匹配 |
| AI自动分级 | `autoGradeImage()` | 基于场景内容自动调整，支持10种风格、8种场景类型 |
| 风格迁移 | `transferStyle()` | 参考图像风格应用，支持色彩映射和直方图匹配 |
| 色彩校正 | `applyColorCorrection()` | 应用亮度/对比度/饱和度/色温/色调/伽马/Lift/Gamma/Gain |

### 2. 场景理解增强 (`ai/scene-understanding.ts`)

| 功能 | 函数 | 说明 |
|------|------|------|
| 物体检测 | `detectObjects()` | 基于颜色分割和轮廓检测，支持11种物体类别 |
| 人脸检测 | `detectFaces()` | 基于肤色检测和椭圆拟合，含关键点/表情/头部姿态 |
| 动作识别 | `recognizeActions()` | 基于帧间运动分析，支持运动分类 |
| 语义分割 | `segmentSemantics()` | 基于位置和颜色的场景分割，8种语义类别 |
| 场景描述 | `describeScene()` | 综合场景分析，含类型/氛围/光照/时间段 |
| 物体跟踪 | `trackObjects()` | IoU匹配的跨帧物体跟踪 |

### 3. 智能剪辑增强 (`ai/smart-editing.ts`)

| 功能 | 函数 | 说明 |
|------|------|------|
| 节拍检测 | `detectBeats()` | 音频能量包络分析，检测BPM和节拍点 |
| 情绪分析 | `analyzeEmotion()` | 音频特征推断情绪，含高潮点检测 |
| 剪辑建议 | `generateCutSuggestions()` | 节拍/情绪/场景变化综合剪辑点建议 |
| 预告片生成 | `generateTrailer()` | 自动预告片，支持5种风格、4种节奏 |
| 智能排序 | `sortSegments()` | 6种排序策略（叙事/情绪/节奏/重要性等） |
| 节奏匹配 | `rhythmMatchEdit()` | 根据音频节拍自动剪辑 |
| 情绪感知 | `emotionAwareEdit()` | 根据情绪变化建议剪辑点 |

### 4. ACES色彩管理 (`color/aces.ts`)

| 功能 | 说明 |
|------|------|
| 色彩空间转换 | sRGB/Rec.709/Rec.2020/ACES2065-1/ACEScg/ACEScct 互转 |
| 传输特性 | sRGB/Gamma2.2/PQ/HLG/ACEScct 线性化和编码 |
| 色调映射 | Reinhard/ACES Hill/ACES Narkowicz/Filmic/AGX |
| LUT管理 | 1D/3D LUT加载、应用（三线性插值）、.cube文件解析 |
| ACES矩阵 | AP0/AP1/Rec.709/Rec.2020 转换矩阵完整实现 |
| OCIO配置 | 自动生成OpenColorIO配置文件 |

## 📐 架构设计原则

1. **本地优先** — 所有AI推理在本地完成，不依赖云端API
2. **纯计算函数** — 无副作用，易于测试和复用
3. **TypeScript类型安全** — 完整的接口定义和类型约束
4. **配置驱动** — 所有模块支持可配置参数和合理默认值
5. **验证/归一化** — 每个参数类型提供 `createDefault`/`validate`/`normalize` 函数三件套
6. **ACES标准** — 遵循ACES色彩管理标准，确保色彩准确性
