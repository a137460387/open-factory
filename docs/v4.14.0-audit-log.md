## 点 1 — RUSTSEC-2026-0195

### 结论（已更正）

> **更正**：审计初稿声称"quick-xml 0.37.5 DoS 漏洞已修复"和"Cargo audit 在 HEAD 上通过，无新漏洞"——**这是错误的**。
> 审计在 2026-06-29 advisory 发布前执行，当时 quick-xml 0.39.4 确实通过了 audit。
> 06-29 新发布两条 HIGH advisory 后，0.39.4 不再满足安全要求。

CI `cargo audit` 实际输出为：**2 vulnerabilities + 1 warning**。

#### Vulnerability 1 — RUSTSEC-2026-0194 (HIGH, CVSS 7.5)

- **Crate:** quick-xml 0.39.4
- **Title:** Quadratic run time when checking a start tag for duplicate attribute names
- **Date:** 2026-06-29
- **Issue:** O(N²) complexity when checking for duplicate attribute names in XML start tags. An attacker can craft a tag with many attributes to cause CPU exhaustion.
- **Fix:** Upgrade to `>= 0.41.0`

#### Vulnerability 2 — RUSTSEC-2026-0195 (HIGH, CVSS 7.5)

- **Crate:** quick-xml 0.39.4
- **Title:** Unbounded namespace-declaration allocation in NsReader enables memory-exhaustion DoS
- **Date:** 2026-06-29
- **Issue:** `NsReader` performs unbounded heap allocation for namespace declarations on start tags, allowing an attacker to OOM a process with a crafted XML tag.
- **Fix:** Upgrade to `>= 0.41.0`

#### Warning — RUSTSEC-2026-0190 (Unsound)

- **Crate:** anyhow 1.0.102
- **Title:** Unsoundness in `Error::downcast_mut()`
- **Date:** 2026-06-25
- **Issue:** Violates Rust's borrow rules (Stacked Borrows model), resulting in undefined behavior.
- **Fix:** Upgrade to `>= 1.0.103`

#### audit.toml 状态

`apps/desktop/src-tauri/.cargo/audit.toml` 有 18 个 ignore 条目（unic-\*、gtk-rs、proc-macro-error 等），
**没有** RUSTSEC-2026-0194、0195、0190 的 ignore。CI 失败的直接原因。

#### 升级路径受阻

```
quick-xml 0.39.4 ← plist 1.9.0 ← tauri / tauri-build / tauri-codegen / tauri-utils
```

无法直接控制 plist/tauri 的版本约束。需等待上游发布新版本，或临时 ignore。

## 已知 CI 问题（超出本次审计范围）

以下四个 CI 失败均与本轮死代码清理无关，不阻塞本次清理的合并。

### 1. RUSTSEC-2026-0194/0195 (quick-xml)

- **原因：** 2026-06-29 新发布 advisory，要求 quick-xml ≥ 0.41.0，当前 0.39.4
- **升级受阻：** quick-xml ← plist 1.9.0 ← tauri，需等上游发布
- **建议：** 临时在 audit.toml 添加 ignore（附注释追踪），或等 tauri 更新

### 2. RUSTSEC-2026-0190 (anyhow)

- **原因：** anyhow 1.0.102 的 `Error::downcast_mut()` unsoundness
- **修复：** `cargo update -p anyhow`（若 semver 允许升到 1.0.103+）

### 3. e2e Vite 解析失败

- **原因：** CI e2e job 加入后从未成功。Playwright webServer 启动 Vite dev server 时，
  Vite 尝试解析 `@open-factory/editor-core` 的 `exports` → `./dist/index.js`，
  但 `dist/` 从未 build（gitignore）。CI workflow 缺少 workspace packages 的 build 步骤。
- **首次失败：** commit 4a7f2c59（"ci: 新增 E2E 测试 job 到 CI workflow"）加入 e2e job 后的第一次运行
- **修复方案 A（推荐）：** `apps/desktop/vite.config.ts` 加 `resolve.alias` 映射 workspace 包到 source
  ```typescript
  resolve: {
    alias: {
      '@open-factory/editor-core': path.resolve(__dirname, '../../packages/editor-core/src/index.ts'),
      '@open-factory/plugin-sdk': path.resolve(__dirname, '../../packages/plugin-sdk/src/index.ts'),
    }
  }
  ```
- **修复方案 B：** CI e2e job 前加 build 步骤
  ```yaml
  - name: Build workspace packages
    run: |
      tsc -p packages/editor-core/tsconfig.build.json
      tsc -p packages/plugin-sdk/tsconfig.build.json
  ```

### 4. frontend locale 测试失败

- **原因：** `ShortcutCheatsheetPanel.test.tsx` 期望中文字符串，CI runner `navigator.language=en-US` 渲染英文
- **性质：** 测试 bug，非产品 bug
- **修复：** 测试 `beforeAll` 中加 `setLanguage('zh')`
