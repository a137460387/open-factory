# v4.26.0 迁移指南

**版本:** v4.26.0
**日期:** 2026-09-12
**适用对象:** 插件开发者、内部开发者、外部贡献者

---

## 概述

v4.26.0 完成了大规模架构重构，将超大文件和 God Store 拆分为模块化结构。**所有旧的 import 路径保持向后兼容**（通过 barrel re-export），但推荐迁移到新的模块化路径以优化 Tree-shaking 和开发体验。

---

## 1. Store 迁移

### 1.1 editorUIStore 拆分

旧的 `editorUIStore` 中的 UI 状态已拆分为 4 个独立 Store：

| 旧路径 | 新路径 | 说明 |
|--------|--------|------|
| `useEditorStore` (对话框状态) | `useDialogStore` | 对话框开关状态 (~25 个) |
| `useEditorStore` (模态框状态) | `useModalStore` | 模态弹窗状态 (~15 个) |
| `useEditorStore` (面板状态) | `usePanelStore` | 面板可见性与布局 (~15 个) |
| `useEditorStore` (工具栏状态) | `useToolbarStore` | 工具栏/菜单状态 (~10 个) |

**Before:**

```tsx
import { useEditorStore } from '@/store/editorStore';

function MyComponent() {
  const showExportDialog = useEditorStore((s) => s.showExportDialog);
  const setShowExportDialog = useEditorStore((s) => s.setShowExportDialog);
  // ...
}
```

**After:**

```tsx
import { useDialogStore } from '@/store/dialogStore';

function MyComponent() {
  const showExportDialog = useDialogStore((s) => s.showExportDialog);
  const setShowExportDialog = useDialogStore((s) => s.setShowExportDialog);
  // ...
}
```

### 1.2 editorFeatureStore 拆分

旧的 `editorFeatureStore`（God Store）已按功能域拆分：

| 旧路径 | 新路径 | 说明 |
|--------|--------|------|
| `useEditorFeatureStore` (AI 状态) | `useAiFeatureStore` | AI 相关功能状态 (~15 个) |
| `useEditorFeatureStore` (导出状态) | `useExportFeatureStore` | 导出相关状态 (~10 个) |
| `useEditorFeatureStore` (时间线功能) | `useTimelineFeatureStore` | 时间线增强功能 (~10 个) |
| `useEditorFeatureStore` (媒体功能) | `useMediaFeatureStore` | 媒体管理功能 (~10 个) |

**Before:**

```tsx
import { useEditorFeatureStore } from '@/store/editorFeatureStore';

function MyComponent() {
  const aiProvider = useEditorFeatureStore((s) => s.aiProvider);
  const exportProgress = useEditorFeatureStore((s) => s.exportProgress);
  // ...
}
```

**After:**

```tsx
import { useAiFeatureStore } from '@/store/aiFeatureStore';
import { useExportFeatureStore } from '@/store/exportFeatureStore';

function MyComponent() {
  const aiProvider = useAiFeatureStore((s) => s.aiProvider);
  const exportProgress = useExportFeatureStore((s) => s.exportProgress);
  // ...
}
```

## 4. 插件开发者注意事项

### 4.1 插件 API 兼容性

`packages/plugin-sdk` 的公共 API **未发生变化**，插件无需修改。

### 4.2 推荐的 Import 策略

1. **插件代码:** 继续使用 `@open-factory/plugin-sdk` 的公共 API
2. **内部开发:** 推荐直接引用具体模块路径以优化 Tree-shaking
3. **测试代码:** 可继续使用旧路径（barrel re-export）

### 4.3 Tree-shaking 优化建议

```tsx
// ❌ 不推荐 — 导入整个 barrel
import { buildFfmpegExportPlan, normalizeExportSettings, buildVisualFilters } from '@open-factory/editor-core/export/ffmpeg-builder';

// ✅ 推荐 — 按需导入具体模块
import { buildFfmpegExportPlan } from '@open-factory/editor-core/export/ffmpeg-builder/export-plan';
import { normalizeExportSettings } from '@open-factory/editor-core/export/ffmpeg-builder/settings-normalize';
import { buildVisualFilters } from '@open-factory/editor-core/export/ffmpeg-builder/visual-filters';
```

## 5. 常见问题

### Q1: 旧的 import 路径会报错吗？

**A:** 不会。所有旧路径通过 barrel re-export 保持向后兼容。但推荐迁移到新路径以获得更好的 Tree-shaking 支持。

### Q2: 我需要一次性迁移所有代码吗？

**A:** 不需要。旧路径将持续可用。建议在修改相关文件时逐步迁移。

### Q3: 如何确认我的插件是否兼容？

**A:** 运行 `bun run typecheck` 和 `bun run test` 确认无新增错误。插件 SDK 的公共 API 未变化。

### Q4: 性能有提升吗？

**A:** 是的。主要提升包括：
- Timeline/Inspector 作为独立 lazy chunk，首屏加载更快
- Store 按功能域拆分后，状态更新更精确，减少不必要重渲染
- 模块化后支持更细粒度的 Tree-shaking
