# Open Factory v4.52.1 Release Notes

**发布日期**: 2026-07-21

**基于审计**: v4.52.0 全量系统审计

## 🛠️ P0 紧急修复

### 1. TaskScheduler 时间片超时误判修复

**问题**: `TIME_SLICE_EXCEEDED` 错误被误判为任务失败，导致任务被无效重试最多 3 次。
**影响**: 任何超过 16ms 的任务都会被错误地执行 4 次，浪费计算资源。
**修复**: 区分调度信号（`TimeSliceExceeded`）与真实任务失败，时间片超时不计入重试次数。

**文件**: `packages/editor-core/src/core/task-scheduler.ts`

### 2. 帧缓存 MinHeap 性能优化

**问题**: `pruneToBudget()` 每次写入触发 O(n log n) 全量排序，高频帧写入时成为瓶颈。
**影响**: 快速播放场景下可能导致帧丢失。
**修复**: 引入 MinHeap 数据结构，淘汰复杂度降至 O(log n)。添加 `reheapify()` 方法处理优先级动态更新。

**文件**: `packages/editor-core/src/render-cache.ts`, `apps/desktop/src/lib/preview/gpu-acceleration.ts`

### 3. 任务饥饿预防机制

**问题**: 被抢占的低优先级任务可能因高优先级任务持续涌入而永远无法完成。
**修复**: 被抢占 3 次后临时提升优先级至 `normal`，防止 starvation。

**文件**: `packages/editor-core/src/core/task-scheduler.ts`

---

## 🛡️ P1 改进

### 4. AI Worker Transferable 零拷贝优化

**改进**: `AIWorkerPool.dispatchToWorker()` 为 `Float32Array` 输入添加 Transferable Objects 支持，避免 structured clone 拷贝大型张量数据。

**文件**: `packages/editor-core/src/ai/ai-worker.ts`

### 5. GPU 缓存字节级内存限制

**问题**: `GPUColorProcessor` 缓存仅按条目数限制（64帧），4K 场景下可达 ~2GB 无界内存。
**修复**: 新增 `maxCacheBytes` 配置（默认 512MB），改为字节级 LRU 淘汰，同时保留条目数限制。

**文件**: `packages/editor-core/src/color/gpu-color-processing.ts`
