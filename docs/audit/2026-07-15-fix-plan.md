## 一、紧急修复（Critical + 安全 High）

### C1: WebDAV 密码加密密钥可预测

| 项目 | 内容 |
|------|------|
| **文件** | `apps/desktop/src-tauri/src/commands/backup.rs:507-512` |
| **问题** | `derive_secret_key` 使用 SHA-256(固定字符串 + app_dir) 派生密钥，密钥材料完全可预测 |
| **影响** | 攻击者可读取 `backup-secrets.json` 后解密所有 WebDAV 密码 |
| **修复方案** | 迁移到系统 keyring，复用 `secrets.rs` 的 `Entry::new(service, account)` 模式 |
| **具体步骤** | 1. 新增 `WEBDAV_KEYCHAIN_SERVICE` 常量<br>2. 为 3 组密码各创建 keyring entry<br>3. 保留旧文件解密能力用于迁移<br>4. 读取时优先 keyring，回退到旧文件 |
| **负责人** | 落小雨 |
| **截止时间** | 立即 |

### C2: 中文分词完全失效

| 项目 | 内容 |
|------|------|
| **文件** | `packages/editor-core/src/ai-speech-understanding.ts:250-256` |
| **问题** | `tokenize` 按空白分割，中文不含空格导致整段文本被当作一个 token |
| **影响** | 关键词提取、话题聚类、摘要生成功能对中文完全失效 |
| **修复方案** | 实现 n-gram 分词（bigram + unigram 混合策略），无需外部依赖 |
| **具体步骤** | 1. 中文字符序列按 bigram 切分<br>2. 英文/数字仍按空白分割<br>3. 合并结果并去重<br>4. 添加对应单元测试 |
| **负责人** | 落小雨 |
| **截止时间** | 立即 |

### H1: WebDAV nonce 可预测

| 项目 | 内容 |
|------|------|
| **文件** | `apps/desktop/src-tauri/src/commands/backup.rs:514-527` |
| **问题** | nonce 由 SystemTime + PID + password.len() 派生，可预测且有复用风险 |
| **影响** | AES-GCM nonce 复用会导致认证完全失效 |
| **修复方案** | 使用 `OsRng` 生成密码学安全随机 nonce（随 C1 一起修复） |
| **负责人** | 落小雨 |
| **截止时间** | 随 C1 一起 |

### H2: Asset Protocol scope 过宽

| 项目 | 内容 |
|------|------|
| **文件** | `apps/desktop/src-tauri/tauri.conf.json:26-35` |
| **问题** | 允许访问 `$APPDATA/**`、`$DOCUMENT/**`、`$DOWNLOAD/**` 等广泛目录 |
| **影响** | XSS 存在时攻击者可读取任意用户文件 |
| **修复方案** | 仅保留 `$APPDATA/**` 和 `$APPCACHE/**`，其余目录通过运行时动态授权添加 |
| **负责人** | 落小雨 |
| **截止时间** | 立即 |
