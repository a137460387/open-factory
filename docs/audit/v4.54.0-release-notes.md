## 智能模板应用

### 1. 内容感知适配引擎

**文件**: `packages/editor-core/src/ai/template-adapter.ts`

自动分析素材特征（时长、画面复杂度、音频频谱），将分析结果映射为模板参数调整：

- **时长适配**: 按比例缩放 `flexibleDuration` 片段，限制 0.5x-2.0x 范围
- **画面适配**: 低复杂度素材降低效果强度 (0.4x) 避免过度处理，高复杂度素材降低效果 (0.75x) 避免视觉噪音
- **音频适配**: 匹配语音音量、调整 ducking 强度、对齐 beat 间隔的 fade 曲线
- **一键智能适配**: `createSmartAdaptation(project, template)` 自动选取主素材 (video > image > audio)

### 2. 自动节奏匹配（踩点）

**文件**: `packages/editor-core/src/ai/rhythm-matcher.ts`

音频节拍检测与视频动态性分析，将节奏点与模板关键帧对齐：

- **节拍检测**: 基于 RMS 能量峰值检测，窗口化计算 → 归一化 → 局部峰值筛选 → 近距去重
- **视频动态分析**: 归一化帧间运动幅度，通过前后帧梯度估算方向
- **踩点效果**: 在节拍点注入 scale-punch / opacity-pulse / position-shift 关键帧
- **一键对齐**: `createRhythmAlignedTemplate()` 合并音频节拍与视频运动峰值

### 3. 模板推荐引擎

**文件**: `packages/editor-core/src/ai/template-recommender.ts`

基于项目内容与用户历史，使用加权余弦相似度推荐适配模板：

- **内容匹配 (40%)**: 时长、类别、节奏匹配
- **用户偏好 (30%)**: 历史类别/节奏/转场亲和度
- **素材适配 (30%)**: 对话、运动、素材类型、音乐风格
- **Top-K 推荐**: `recommendTemplates()` 评分排序，`explainRecommendation()` 生成推荐理由

## 协作功能实战

### 4. WebRTC NAT 穿透与 TURN 中继

**文件**: `packages/editor-core/src/collaboration/webrtc-connection.ts`

完整 WebRTC P2P 连接优化：

- **ICE 候选交换**: 完整 ICE 候选收集与交换流程
- **TURN 集成**: 可配置 TURN 服务器，支持 NAT 穿透
- **DataChannel**: 用于传输 CRDT 操作数据
- **自动重连**: 指数退避重连机制（复用 `ws-transport.ts` 的 `computeBackoffDelay`）
- **连接监控**: 连接状态变更回调与统计信息

### 5. CRDT 冲突解决策略验证

**文件**: `packages/editor-core/src/collaboration/conflict-resolver.ts`

时间线操作的 CRDT 语义定义与并发操作合并验证：

- **向量时钟**: `createVectorClock()` / `incrementClock()` 因果序比较
- **冲突检测**: `detectTimelineConflict()` 检测并发 + 同目标操作
- **解决策略**: last-writer-wins / first-writer-wins / merge / manual
- **批处理**: `batchOperations()` 时间窗口滑动批处理高频操作
- **ConflictResolver 类**: 追踪操作、自动解决、历史记录

### 6. 离线编辑队列与自动合并

**文件**: `packages/editor-core/src/collaboration/offline-support.ts`

本地操作队列与在线/离线状态管理：

- **操作队列**: `OfflineOperationQueue` 支持 enqueue/dequeue/markSynced/getStats
- **持久化抽象**: `OfflineStorageAdapter` 接口，默认 `MemoryStorageAdapter`，可替换 IndexedDB
- **批量同步**: 每批最多 50 个操作，跳过 ephemeral awareness 操作
- **状态管理**: `OfflineSyncManager` 自动检测 online/offline 事件触发同步
- **冲突解决 UI**: `ConflictResolutionUI` 接口支持 local/remote/merge 三种解决方式
