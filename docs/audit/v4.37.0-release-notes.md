# Open Factory v4.37.0 — AI 说话人分离与 TTS 语音合成

> 🎉 **里程碑：完整本地 AI 音频工作流闭环达成**

## 🎙️ AI 说话人分离

基于本地 Pyannote 模型的说话人分离系统，完全离线运行，自动区分不同说话人并分配标签。

| 功能 | 描述 |
|------|------|
| **嵌入向量提取** | 基于 Pyannote 模型提取音频帧级嵌入向量，支持 MFCC 特征降级 |
| **聚类算法** | 自适应 K-Means 聚类，自动确定最优说话人数量（2-8 人） |
| **说话人标签分配** | 自动生成 Speaker 1/2/3 等标签，支持自定义命名 |
| **置信度评估** | 每个分离结果附带置信度分数，低置信度区域自动标记 |
| **转录集成** | 与 Whisper 转录结果无缝对接，自动为每段字幕分配说话人 ID |
| **多机位联动** | 基于说话人变化自动生成多机位切换建议，遵守最小间隔阈值 |

**核心文件**:
- `packages/editor-core/src/ai/speaker-diarization.ts` — 说话人分离核心算法（430 行）
- `apps/desktop/src/workers/ai-speaker-diarization.worker.ts` — Web Worker 隔离推理
- `apps/desktop/src/components/AI/SpeakerDiarizationPanel.tsx` — UI 面板
- `apps/desktop/src/hooks/useSpeakerDiarization.ts` — 状态管理 Hook

**单元测试**: `packages/editor-core/__tests__/ai-speaker-diarization.test.ts`（29 个用例）

## 🗣️ AI TTS 语音合成

集成 VITS 模型的本地文本转语音系统，支持多语言，自动对齐时间线。

| 功能 | 描述 |
|------|------|
| **多语言支持** | 自动检测文本语言（中/英/日/韩等），选择对应语音模型 |
| **语音库管理** | 内置多语音预设，支持语速、音调、音量参数调节 |
| **智能文本分段** | 按句子边界智能分段，支持标点感知与长度优化 |
| **时间线对齐** | TTS 输出自动对齐原始字幕时间轴，支持时长调整 |
| **音频处理** | 自动归一化、淡入淡出、静音填充，确保音频质量 |
| **配音适配** | 与字幕编辑器深度集成，一键从字幕生成配音轨道 |

**核心文件**:
- `packages/editor-core/src/ai/tts.ts` — TTS 核心算法（520 行）
- `apps/desktop/src/workers/ai-tts.worker.ts` — Web Worker 隔离推理
- `apps/desktop/src/components/AI/TTSPanel.tsx` — UI 面板
- `apps/desktop/src/hooks/useTTS.ts` — 状态管理 Hook

**单元测试**: `packages/editor-core/__tests__/ai-tts.test.ts`（45 个用例）

## 🎬 多机位智能联动

基于说话人变化的多机位切换建议系统。

| 功能 | 描述 |
|------|------|
| **说话人-机位映射** | 自动将说话人分配到不同机位，支持手动调整 |
| **切换建议生成** | 基于说话人变化时间点生成切换建议，遵守最小间隔 |
| **切换策略** | 支持跟随说话人、固定机位、手动模式三种策略 |
| **冲突检测** | 自动检测切换冲突并提供优化建议 |

**核心文件**:
- `apps/desktop/src/components/AI/SpeakerMulticamPanel.tsx` — UI 面板
- `apps/desktop/src/hooks/useSpeakerMulticam.ts` — 状态管理 Hook

## 🏗️ 架构遵循

本版本严格遵循项目架构规范：

- ✅ 核心算法在 `packages/editor-core`，保持 80%+ 覆盖率
- ✅ AI 推理通过 Web Worker 隔离，主线程零阻塞
- ✅ Timeline 突变通过 Command 对象，不直接调用 Zustand setter
- ✅ 所有 Tauri 调用通过 `tauri-bridge.ts` 统一封装
- ✅ 无第三方代码复制，保持 clean-room 实现
- ✅ 无遥测/登录/云服务，保持 local-first 原则
- ✅ 大型 AI 模型推理异步执行，不阻塞 UI
- ✅ E2E 测试保持 `data-testid` 覆盖
