### 4.2 已执行的优化（Sprint G）

1. **Bug 修复**: `SubtitleEditorPanel.handleToggleSelect` — `selectedSet` 从 `useCallback` 依赖移入回调体，修复了 memoization 失效问题
2. **useCallback 移除**（`useEditorShellMediaCallbacks.ts`）:
   - `renameMediaFolder`
   - `deleteMediaFolder`
   - `setMediaFolderCollapsed`
   - `moveMediaToFolder`
   - `handleUpdateSubclip`
   - `handleDeleteSubclip`
   - `jumpToMediaAsset`
   - `updateProjectReleaseVersion`
3. **useCallback 移除**（`useEditorShellTimelineCallbacks.ts`）:
   - `setSpectrumSelectionRange`

### 4.3 React Compiler 自动优化说明

React 19 引入的 React Compiler 会自动：
- 追踪组件内的响应式依赖
- 自动 memoize 没有变化的计算结果
- 保持没有响应式依赖的函数引用稳定

这意味着以下模式中的 `useCallback` 是冗余的：
```tsx
// ❌ 手动 memoization（React 18 模式）
const handleClick = useCallback(() => {
  store.getState().doSomething();
}, []);

// ✅ React Compiler 自动处理（React 19 模式）
const handleClick = () => {
  store.getState().doSomething();
};
```
