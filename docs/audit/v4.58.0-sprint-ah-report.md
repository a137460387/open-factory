## Architecture Decisions

1. **Fastify over Express** - Better TypeScript support, schema validation, performance
2. **Singleton API client** - `initializeApiClient()` + `getApiClient()` pattern for React
3. **Structural typing** - api-client types match plugin-market types exactly
4. **WebGPU fallback** - Automatic degradation: WebGPU → WebGL → CPU
5. **INT8 default** - Best quality/size tradeoff for 14B model

## Known Limitations

1. API Gateway uses mock data (no database yet)
2. WebGPU support depends on browser/engine capabilities
3. Model weights not bundled (downloaded on first use)
4. Plugin-market frontend still uses local `/api/plugins` routes (not yet connected to api-gateway)
