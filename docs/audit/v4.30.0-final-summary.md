```
│       └── EffectsRack (20 种效果)
├── MasterStrip (主输出)
├── MixerConsole (传统混音台视图)
├── SpectrumAnalyzer (频谱分析)
├── ChannelAnalysisPanel (频道分析)
├── AutomationEditor (自动化曲线)
└── NoiseReductionDialog (降噪对话框 lazy loaded)
```

### 状态管理架构

```
mixerStore (Zustand)
├── mixerState: MixerState
│   ├── channels: MixerChannel[]
│   ├── buses: MixerBus[]
│   └── master: MasterBus
├── selectedChannelId: string | null
├── expandedChannelIds: Set<string>
├── activeTab: MixerPanelTab
├── noiseReductionParams: Record<string, NoiseReductionParams>
├── spectrumAnalyzerActive: boolean
└── automationRecordMode: 'read' | 'write' | 'touch' | 'latch'
```

### 降噪管线架构

```
用户交互
    ↓
NoiseReductionDialog
    ├── 预设选择 (light/medium/heavy/custom)
    ├── 强度滑块 (0-100%)
    └── 高级参数编辑
        ↓
noise-reduction.ts
    ├── strengthToNoiseReductionParams() → 参数映射
    ├── normalizeNoiseReductionParams() → 参数校验
    ├── estimateNoiseReduction() → 效果预估
    └── buildNoiseReductionFfmpegArgs() → FFmpeg 参数
        ↓
mixerStore
    └── noiseReductionParams[trackId] → 状态存储
        ↓
audio-filters.ts (导出管线)
    └── buildAfftdnNoiseReductionFilter() → FFmpeg 滤镜
```

## 🔧 技术实现细节

### 1. 多轨混音器实现

#### 核心组件
- **AudioMixer.tsx**: 顶层容器，整合所有子组件，管理标签页切换
- **ChannelStrip.tsx**: 单通道 UI，包含 VU 表、推子、旋钮、按钮
- **MixerConsole.tsx**: 传统混音台视图，水平滚动通道条
- **EffectsRack.tsx**: 效果链管理，支持 20 种效果类型
- **AutomationEditor.tsx**: SVG 自动化曲线编辑器
- **VUMeter.tsx**: 简约 VU 电平表

#### 关键设计决策
1. **延迟加载**: NoiseReductionDialog 和 SpectrumAnalyzer 使用 `React.lazy` 延迟加载
2. **命令模式**: Ducking 操作使用 `BatchUpdateKeyframeCommand` 支持撤销/重做
3. **状态隔离**: 混音器状态独立于编辑器状态，通过 Zustand store 管理
4. **组件复用**: ChannelStrip 在 AudioMixer 和 MixerConsole 中复用

### 2. 频谱分析器实现

#### 技术方案
- **Canvas 渲染**: 使用 HTML5 Canvas 绘制频谱
- **requestAnimationFrame**: 驱动 60fps 动画循环
- **对数频率刻度**: 20 Hz ~ 20 kHz 对数分布
- **峰值保持**: 带衰减动画的峰值指示
- **HiDPI 支持**: 自适应 devicePixelRatio

#### 显示模式
1. **柱状图 (bars)**: 传统频谱显示
2. **线条 (line)**: 连续频谱曲线
3. **填充 (filled)**: 填充频谱区域

### 3. 智能降噪实现

#### 核心算法
- **FFmpeg afftdn**: 基于 FFT 的自适应噪声消除
- **参数映射**: 0-100% 强度线性映射到 -60~0 dB
- **预设系统**: 4 级预设覆盖常见场景

#### 参数体系
```typescript
interface NoiseReductionParams {
  noiseFloor: number;      // 噪声底限 (-60~0 dB)
  nrType: 0 | 1 | 2;      // 降噪类型 (弱/中/强)
  autoNoiseSampling: boolean; // 自动噪声采样
  noiseSampleStart?: number;  // 采样起始时间
  noiseSampleEnd?: number;    // 采样结束时间
}
```

#### 导出集成
- 降噪参数可转换为标准效果槽格式
- 导出时自动注入 FFmpeg afftdn 滤镜
- 支持按轨道独立配置

## 📐 架构遵循情况

### ✅ 已遵循的架构原则

1. **命令模式**: 时间线变更通过命令对象，混音器 Ducking 使用 `BatchUpdateKeyframeCommand`
2. **状态管理**: Zustand store 独立管理混音器状态
3. **组件隔离**: 混音器组件与编辑器组件解耦
4. **延迟加载**: 重量级组件使用 `React.lazy`
5. **FFmpeg 参数数组**: 降噪滤镜使用数组风格参数，不拼接 shell 字符串
6. **本地优先**: 所有音频处理本地执行，无云端依赖
7. **Tauri 桥接**: 所有 Tauri 调用通过 `tauri-bridge.ts`

### ✅ 已遵循的测试规范

1. **Vitest 覆盖**: 所有新模块有 Vitest 测试
2. **覆盖率**: editor-core 覆盖率保持 80%+
3. **data-testid**: UI 组件保留有用的 data-testid
4. **测试隔离**: 测试不依赖外部服务

### ✅ 已遵循的安全规范

1. **无遥测**: 不添加遥测、登录或云服务
2. **本地媒体**: 媒体预览使用 Tauri `convertFileSrc`
3. **FFmpeg 安全**: 使用参数数组，不拼接 shell 字符串
