## 🧩 插件市场 MVP

完整的插件市场系统，支持浏览、搜索、安装和管理插件。

| 功能 | 描述 |
|------|------|
| **插件浏览** | 基于虚拟滚动的高性能插件列表，支持大量插件的流畅浏览 |
| **分类搜索** | 按类别（效果、导出、工作流、AI 模型）筛选，支持自由文本搜索与标签过滤 |
| **详情弹窗** | 完整的插件信息展示，包括描述、权限声明、评分、下载量、版本历史 |
| **一键安装** | 从市场直接安装插件到本地，自动处理依赖与版本兼容性检查 |
| **插件更新** | 自动检测已安装插件的可用更新，支持一键升级 |
| **插件管理** | 统一的插件管理面板，支持启用/禁用、卸载、权限查看 |

**核心文件**:
- `packages/editor-core/src/plugins/plugin-market-service.ts` — 市场服务核心
- `apps/desktop/src/components/PluginMarket/` — UI 组件（4 个组件）

**E2E 测试**: `apps/desktop/e2e/plugin-marketplace.spec.ts`

## 🛡️ 安全与校验

多层安全机制确保插件的完整性与安全性。

| 安全特性 | 描述 |
|----------|------|
| **SHA-256 哈希校验** | 插件包下载后自动验证 SHA-256 哈希，防止篡改与损坏 |
| **权限声明一致性检查** | 验证插件实际使用的权限与声明的权限是否一致，防止权限滥用 |
| **Web Worker 沙箱隔离** | 插件运行在独立的 Web Worker 中，与主线程隔离，确保核心系统稳定 |
| **版本兼容性检查** | 安装前检查插件与当前应用版本的兼容性，避免不兼容问题 |

**核心文件**:
- `packages/editor-core/src/plugins/plugin-market-service.ts` — 哈希校验与验证逻辑
- `packages/editor-core/src/plugins/plugin-manager.ts` — 沙箱隔离与权限管理

## 🛠️ 开发者 CLI 工具

完整的插件开发工具链，降低第三方开发者入门门槛。

| 命令 | 功能 | 说明 |
|------|------|------|
| `create` | 插件脚手架 | 交互式创建插件项目，自动生成目录结构、配置文件与示例代码 |
| `validate` | 插件验证 | 检查插件配置、依赖、权限声明的合法性与完整性 |
| `hash` | 哈希生成 | 为插件包生成 SHA-256 哈希值，用于市场发布时的完整性校验 |
| `debug` | 沙箱调试 | 在沙箱环境中运行插件，支持日志输出与断点调试 |

**核心文件**: `packages/plugin-cli/`（6 个模块）

```
packages/plugin-cli/
├── src/
│   ├── cli.js          # CLI 入口
│   ├── create.js       # 脚手架命令
│   ├── validate.js     # 验证命令
│   ├── hash.js         # 哈希生成命令
│   ├── debug.js        # 沙箱调试命令
│   └── test-runner.js  # 测试运行器
└── index.test.js       # CLI 测试
```

## 🏗️ 架构遵循

本版本严格遵循项目架构规范：

- ✅ 核心算法在 `packages/editor-core`，保持 80%+ 覆盖率
- ✅ Timeline 突变通过 Command 对象，不直接调用 Zustand setter
- ✅ 所有 Tauri 调用通过 `tauri-bridge.ts` 统一封装
- ✅ 插件 SDK 独立包，与核心解耦
- ✅ CLI 工具独立包，便于第三方使用
- ✅ 无第三方代码复制，保持 clean-room 实现
- ✅ 无遥测/登录/云服务，保持 local-first 原则
- ✅ E2E 测试保持 `data-testid` 覆盖
