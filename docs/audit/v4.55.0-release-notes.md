# Open Factory v4.55.0 Release Notes

**Sprint AE — 引擎化与无头渲染**
发布日期：2026-07-22

---

## Highlights

### 🖥️ CLI 引擎

基于 Commander.js 构建的完整命令行框架，将 Open Factory 的核心能力暴露为可编程接口：

| 命令 | 功能 |
|------|------|
| `render` | 无头渲染项目，输出视频/字幕文件 |
| `apply-template` | 批量应用模板到项目 |
| `analyze` | 内容分析（字幕质量、时间轴、翻译） |
| `workflow` | 执行 YAML 工作流编排 |

**源文件：** `packages/cli/src/` — 9 个文件（cli.ts, index.ts, commands/*, core/*）

### ⚙️ 无头运行时 (Headless Runtime)

脱离浏览器 DOM 的纯 Node.js 编辑器核心，支持服务端/CI 环境：

- **HeadlessEditorCore** — 无 DOM 依赖的编辑器实例，直接操作项目数据
- **HeadlessRenderer** — FFmpeg 直接合成管线，跳过 Canvas/WebGPU 渲染
- **HeadlessAnalyzer** — 离线内容分析（字幕质量、翻译一致性、节奏检测）
- **HeadlessAIInference** — 无头 AI 推理引擎，支持 stdin 管道输入
- **TemplateApply** — 模板批量应用引擎

**源文件：** `packages/editor-core/src/headless/` — 6 个文件

### 🐳 环境适配

- **Docker 镜像：** CPU (`Dockerfile`) 和 GPU (`Dockerfile.gpu`) 两种版本
- **Docker Compose：** 一键启动编排配置
- **标准化输出：** JSON 格式输出 + 语义化退出码（0=成功, 1=用户错误, 2=系统错误）
- **工作流编排：** YAML 定义多步骤流水线，支持依赖和条件执行

## Docker 镜像

```bash
# CPU 版本
docker build -f docker/Dockerfile -t open-factory:latest .

# GPU 版本 (NVIDIA CUDA)
docker build -f docker/Dockerfile.gpu -t open-factory:gpu .

# Docker Compose
docker-compose -f docker/docker-compose.yml up
```
