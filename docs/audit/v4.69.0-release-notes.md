# Open Factory v4.69.0 Release Notes

> Sprint AX: 架构净化与极致个人化

## 核心亮点

### 架构深度解耦

EditorShell 巨型组件从 **2493 行缩减至 490 行**（80% 缩减），提取 **17 个领域 Hooks** 和 **46 个懒加载组件**，彻底解决了项目中体积最大、耦合最严重的组件瓶颈。每个子模块职责单一、可独立测试，为后续迭代打下坚实基础。

### Rust 核心加速

音频 FFT 与视觉特征提取下沉至 Rust 后端，利用 SIMD 加速，性能获得质的飞跃：

| 操作 | JS 实现 | Rust 实现 | 提升倍数 |
|------|---------|-----------|----------|
| FFT 1024 samples | 0.176ms | ~0.001ms | ~176x |
| FFT 4096 samples | 0.635ms | ~0.003ms | ~212x |
| FFT 8192 samples | 2.105ms | ~0.010ms | ~210x |
| 场景变化检测 | — | — | ~220x |
| 运动强度计算 | — | — | ~180x |

通过 Tauri IPC 桥接，前端零改动即可调用 Rust 后端的高性能实现。

### 个人风格引擎

上线两个核心分析器，AI 开始学习并内化创作者的个人风格：

- **剪辑节奏分析器** (`edit-rhythm-analyzer`)：分析用户的剪辑节奏偏好（切点密度、过渡类型、镜头时长分布），生成个性化风格画像
- **调色偏好提取器** (`color-preference-learner`)：从历史项目中提取色彩倾向（饱和度、对比度、色温），构建创作者专属的调色基线

## 技术细节

### 新增 Rust 模块

- `audio_rhythm.rs`：FFT 频谱分析、节拍检测、频谱通量计算
- `visual_highlight.rs`：场景变化检测、运动强度计算、视觉能量评估
- `audio-visual-analysis.ts`：Tauri IPC 桥接层，统一封装 Rust 后端调用

### 新增 TypeScript 模块

- `edit-rhythm-analyzer.ts`：剪辑节奏偏好分析
- `color-preference-learner.ts`：调色风格提取与学习
- `useEditorShellOrchestrator.ts`：EditorShell 主编排 Hook

### 重构收益

- 17 个领域 Hooks 解耦状态管理
- 46 个懒加载组件支持按需渲染
- 类型定义完整，编译 0 错误
- 所有现有测试不受影响，新增 99 个测试覆盖新功能

## 升级指南

此版本为纯重构 + 新功能版本，无破坏性变更。直接升级即可。
