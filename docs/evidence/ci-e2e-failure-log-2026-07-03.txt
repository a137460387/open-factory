  "main": "./dist/index.js"

## Contrast with other CI jobs that DON'T fail

  - frontend job: uses `tsc -b` (tsconfig paths resolve from source, not dist/)
  - rust job: includes `bun run build` which runs `tsc -p tsconfig.build.json`
    → produces dist/index.js before anything tries to resolve it
  - e2e job: runs `bunx playwright test` directly, NO build step
    → Playwright webServer starts `vite --host localhost` → dep-scan
    → missing dist/ → dev server crash → all e2e tests fail
