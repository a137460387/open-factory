# v4.27.0 产品规划提案

**提案日期**: 2026-07-16
**提案人**: 落小雨
**状态**: 📋 待评审
**前置版本**: v4.26.0（架构重构完成）

---

## 背景

v4.26.0 完成了 Open Factory 历史上最大规模的架构重构：Store 拆分、超大组件瘦身、逻辑模块化。当前架构健康度极高（editor-core 覆盖率 97.94%、5,258 测试全通过），为产品功能迭代提供了坚实基础。

v4.27.0 应从"架构优化"转向"产品价值驱动"，聚焦用户体验提升和功能增强，让用户直观感受到编辑体验的飞跃。

---

## 候选 Sprint 主题

### 🎯 Sprint A：智能粗剪面板 (Smart Rough-Cut Panel)

**优先级建议**: ⭐⭐⭐⭐⭐ 最高
**用户价值**: 将 AI 能力串联为完整工作流，大幅缩短从素材到粗剪的时间

**功能描述**:
整合已有的 AI 能力（场景检测、静音检测、Whisper 字幕、节拍检测）为一个统一的智能粗剪面板，实现"一键粗剪"：

1. **智能分析阶段** — 自动执行场景检测 + 静音检测 + 语音转文字
2. **AI 编排阶段** — 根据分析结果自动生成剪辑建议（删除静音段、保留高能片段、按叙事逻辑排序）
3. **交互式调整** — 用户在面板中预览 AI 建议，拖拽调整顺序，一键应用到时间线
4. **剪辑报告** — 生成包含素材分析、剪辑决策、时长统计的结构化报告

**技术要点**:
- 复用已有的 `scene-detection`、`silence-detection`、`whisper-subtitle` 模块
- 新增 `SmartRoughCutPanel` 组件，遵循 v4.26.0 模块化架构
- AI 编排逻辑放在 `packages/editor-core` 中，保持纯函数可测试性
- 面板使用 lazy chunk 加载

**预估工期**: 2-3 周
**风险**: 中（AI 编排逻辑复杂度较高，需要充分测试）

---

### 🎯 Sprint B：导出性能与体验升级

**优先级建议**: ⭐⭐⭐⭐ 高
**用户价值**: 导出是用户最常等待的操作，性能提升直接感知明显

**功能描述**:

1. **WASM/GPU 加速预览** — 将 WebGL 预览管线中的关键滤镜迁移至 WASM，减少主线程阻塞
2. **进度预览增强** — 导出过程中实时显示已渲染帧的缩略图预览，而非仅进度条
3. **智能导出调度** — 根据项目复杂度自动选择最优编码参数（码率、preset、线程数）
4. **VMAF 实时质量监控** — 导出过程中实时计算 VMAF 分数，低于阈值时自动提示
5. **批量导出优化** — 多项目批量导出时共享 FFmpeg 进程池，减少启动开销

**技术要点**:
- 修改 `ffmpeg-builder` 模块（v4.26.0 已拆分为 8 个子模块，改动影响面可控）
- 新增 `ExportPreviewCanvas` 组件用于实时帧预览
- 导出调度逻辑新增 `export-scheduler.ts`

**预估工期**: 2 周
**风险**: 低（增量改进，不涉及架构变更）

---

### 🎯 Sprint C：转场效果库与关键帧增强

**优先级建议**: ⭐⭐⭐⭐ 高
**用户价值**: 丰富的转场效果是专业视频编辑器的核心竞争力

**功能描述**:

1. **内置转场效果库** — 新增 15+ 专业转场效果
   - 基础：交叉溶解、推拉、擦除、缩放
   - 进阶：光线泄漏、胶片烧伤、动态模糊、故障风
   - 3D：翻转、立方体旋转、门户穿越
2. **转场预览面板** — 可视化浏览所有转场效果，鼠标悬停即时预览
3. **一键应用** — 从预览面板拖拽到时间线片段之间即可应用
4. **关键帧曲线编辑器增强** — 支持贝塞尔手柄可视化拖拽、缓动预设库（30+ 缓动曲线）
5. **关键帧复制/粘贴** — 支持跨片段复制关键帧动画

**技术要点**:
- 转场效果通过 WebGL shader 实现，预览用 canvas，导出用 FFmpeg xfade
- 新增 `TransitionLibrary` 组件和 `transition-presets/` 模块
- 关键帧曲线编辑器增强现有 `KeyframeEditor` 组件

**预估工期**: 3 周
**风险**: 中（3D 转场 shader 开发复杂度较高）

---

### 🎯 Sprint D：React 19 + Tailwind CSS 4 现代化升级

**优先级建议**: ⭐⭐⭐ 中
**用户价值**: 间接提升（性能改善、开发效率提升），但对用户无直接功能感知

**功能描述**:

1. **React 19 升级** — 利用 React Compiler 自动 memo、Server Components（未来）、use() hook
2. **Tailwind CSS 4 迁移** — 利用新的 CSS-first 配置、氧化引擎（Oxide）、更快的编译速度
3. **依赖链清理** — 移除 React 18 兼容层、清理过时的 polyfill
4. **Bundle 体积优化** — 利用 React 19 的 tree-shaking 改进进一步减小打包体积

**技术要点**:
- 需要全面回归测试所有组件
- Zustand 需要验证与 React 19 的兼容性
- Tailwind CSS 4 的配置格式变化较大，需要逐组件迁移

**预估工期**: 2 周
**风险**: 中高（大版本升级可能引入意外 breaking change）

---

### 🎯 Sprint E：插件生态系统增强

**优先级建议**: ⭐⭐⭐ 中
**用户价值**: 增强扩展性，吸引社区贡献者

**功能描述**:

1. **插件市场 UI** — 在应用内添加插件浏览/安装/更新界面
2. **插件 API v2** — 扩展插件能力，新增：
   - `onTimelineChange` — 监听时间线变更事件
   - `onExportBefore` / `onExportAfter` — 导出前后钩子（已有，增强）
   - `registerPanel` — 注册自定义面板
   - `registerTool` — 注册自定义工具
3. **插件沙箱增强** — Worker 隔离 + 资源限制 + 权限声明
4. **示例插件集** — 提供 3-5 个高质量示例插件作为开发参考
5. **插件开发文档** — 完善插件开发指南和 API 参考

**技术要点**:
- 插件市场需要本地索引 + 远程注册表（可选）
- 插件 API v2 需要向后兼容 v1
- 沙箱增强涉及 Tauri Worker 隔离策略

**预估工期**: 3 周
**风险**: 中（插件 API 设计需要仔细考虑向后兼容）

## 推荐优先级排序

| 优先级 | Sprint | 理由 |
|--------|--------|------|
| 🥇 P0 | A: 智能粗剪面板 | 最大化已有 AI 投资回报，用户感知最强 |
| 🥈 P1 | B: 导出性能升级 | 用户高频操作，性能提升直接可感知 |
| 🥉 P2 | C: 转场效果库 | 差异化竞争力，专业编辑器必备 |
| P3 | E: 插件生态增强 | 长期价值，但短期用户感知较弱 |
| P4 | D: React 19 升级 | 技术债清理，建议在功能迭代间穿插 |

---

## 建议的 v4.27.0 范围

基于 4-6 周的迭代周期，建议选取：

- **Sprint A（智能粗剪面板）** — 核心功能，2-3 周
- **Sprint B（导出性能升级）** — 增量改进，2 周
- 两者并行开发，互不阻塞

Sprint C/D/E 可安排到 v4.28.0 或后续版本。

---

## 技术约束提醒

根据 AGENTS.md 和 roadmap.md 的约束：

- ✅ 所有新功能必须遵循 v4.26.0 的模块化架构（独立 Store、子组件、lazy chunk）
- ✅ 核心算法必须有 Vitest 覆盖，editor-core 保持 ≥ 80%（当前 97.94%）
- ✅ Timeline/Inspector 相关改动需保持 `data-testid` 覆盖
- ✅ FFmpeg 相关改动需使用参数数组风格，不执行 shell 字符串
- ✅ 所有 Tauri 调用走 `tauri-bridge.ts`
- ✅ 大文件处理保持异步，不阻塞 UI 线程
