# 创作者入驻指南

> 本指南将帮助你快速完成 Open Factory 创作者激励计划的入驻流程。

## 入驻前准备

### 账号要求

- 已注册 Open Factory 账号
- 账号状态正常，无违规记录
- 已完成邮箱验证

### 实名认证材料

- 个人创作者：身份证正反面照片
- 企业创作者：营业执照扫描件 + 法人身份证
- 海外创作者：护照或其他有效证件

### 技术准备

- 熟悉基本的编程概念（插件创作）
- 了解 JSON/YAML 配置格式
- 准备好开发环境（推荐 Node.js 18+）

## 入驻流程

### 第一步：申请入驻

1. 登录 Open Factory 控制台
2. 进入「创作者中心」→「入驻申请」
3. 阅读并同意《创作者激励计划协议》
4. 填写基本信息：
   - 创作者昵称
   - 联系邮箱
   - 专业领域（可多选）
   - 个人简介

### 第二步：实名认证

1. 进入「账号设置」→「实名认证」
2. 选择认证类型（个人/企业）
3. 上传认证材料
4. 等待审核（通常 1-3 个工作日）

### 第三步：开发者资质

1. 进入「创作者中心」→「开发者设置」
2. 完成开发者知识测试（20 道选择题，80 分及格）
3. 签署《开发者协议》
4. 获取开发者 API Key

### 第四步：发布首个作品

1. 创建你的第一个插件或工作流
2. 完成测试和文档
3. 提交审核
4. 审核通过后自动激活创作者身份

## 开发者知识测试

测试内容涵盖以下领域：

### 插件开发基础（40%）

- 插件生命周期
- API 接口规范
- 事件处理机制
- 错误处理最佳实践

### 安全规范（30%）

- 数据安全要求
- 权限管理原则
- 敏感信息处理
- 漏洞防范意识

### 平台规则（30%）

- 内容审核标准
- 知识产权规定
- 违规处罚条例
- 用户隐私保护

### 测试样题

```
Q: 插件在处理用户数据时，以下哪种做法是正确的？
A. 将用户数据存储在插件本地缓存
B. 将用户数据加密后存储，并提供删除接口 ✓
C. 将用户数据发送到第三方服务
D. 不需要特殊处理

Q: 当插件检测到安全漏洞时，应该？
A. 立即公开披露
B. 忽略，等待用户反馈
C. 通过安全渠道报告给平台安全团队 ✓
D. 自行修复后发布
```

## 首个作品发布指南

### 选择创作类型

| 类型 | 难度 | 适合人群 |
|------|------|---------|
| 模板插件 | 入门 | 初学者，快速上手 |
| 工具插件 | 中等 | 有编程基础的开发者 |
| 集成插件 | 进阶 | 熟悉第三方 API 的开发者 |
| 工作流 | 中等 | 了解业务流程的用户 |

### 推荐首个作品

对于新手创作者，推荐从以下类型开始：

1. **模板插件**：创建一个简单的模板生成器
2. **格式转换工具**：实现文件格式转换功能
3. **数据处理工具**：简单的数据清洗或转换工具

### 质量标准

首个作品需满足以下标准：

- 功能完整，可正常使用
- 包含基本文档和使用说明
- 无明显 Bug
- 通过安全审核

## 审核流程

### 审核时长

- 普通审核：1-3 个工作日
- 加急审核：24 小时内（需申请）

### 审核内容

1. **功能完整性**：功能是否正常工作
2. **代码质量**：代码规范和可维护性
3. **安全性**：是否存在安全隐患
4. **文档质量**：是否有清晰的使用说明
5. **合规性**：是否符合平台规则

### 审核结果

- **通过**：作品自动上架，创作者身份激活
- **需修改**：提供修改意见，修改后可重新提交
- **不通过**：说明原因，可申诉或重新创作

## 创作者权益激活

审核通过后，以下权益自动激活：

- 创作者后台数据面板
- 收益统计功能
- 基础创作工具使用权限
- 创作者社区访问权限

## 常见入驻问题

### Q: 实名认证审核需要多久？

A: 通常 1-3 个工作日，高峰期可能延长至 5 个工作日。

### Q: 开发者测试可以重考吗？

A: 可以，每次测试间隔需大于 24 小时，最多可考 3 次/月。

### Q: 首个作品必须是插件吗？

A: 不必须，工作流也可以作为首个作品。

### Q: 入驻后可以更换创作者昵称吗？

A: 可以，每 90 天可修改一次。

### Q: 企业入驻和个人入驻有什么区别？

A: 企业入驻可开具增值税专用发票，享受企业级技术支持，但需提供更多资质材料。
