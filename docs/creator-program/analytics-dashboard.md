# 数据分析仪表盘设计文档

> 创作者数据分析仪表盘为创作者提供全面、实时的数据洞察，帮助创作者了解作品表现并优化运营策略。

## 仪表盘概述

### 设计目标

- **实时性**：关键数据实时更新，延迟不超过 5 分钟
- **全面性**：覆盖下载量、活跃用户、评分、收入等核心指标
- **可操作性**：数据可视化配合行动建议
- **易用性**：直观的界面设计，降低数据理解门槛

### 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    前端展示层 (React)                         │
├─────────────────────────────────────────────────────────────┤
│  图表库 (ECharts/Recharts)  │  组件库 (shadcn/ui)           │
├─────────────────────────────────────────────────────────────┤
│                    数据服务层 (API)                           │
├─────────────────────────────────────────────────────────────┤
│  实时计算 (Redis)  │  数据仓库 (ClickHouse)  │  缓存 (CDN)  │
├─────────────────────────────────────────────────────────────┤
│                    数据采集层                                  │
├─────────────────────────────────────────────────────────────┤
│  下载事件  │  用户行为  │  评分数据  │  收入数据               │
└─────────────────────────────────────────────────────────────┘
```

## 核心指标模块

### 1. 下载量统计

#### 数据维度

| 指标 | 说明 | 更新频率 |
|------|------|---------|
| 今日下载量 | 当日实时下载次数 | 实时 |
| 累计下载量 | 历史总下载次数 | 实时 |
| 日均下载量 | 近 30 天日均值 | 每日 |
| 下载增长率 | 环比增长百分比 | 每日 |

#### 可视化设计

```
下载量趋势图
├── 时间范围选择器（7天/30天/90天/自定义）
├── 趋势折线图
│   ├── X 轴：日期
│   ├── Y 轴：下载量
│   └── 辅助线：平均值
├── 对比功能
│   ├── 环比对比（上期同期）
│   └── 同比对比（去年同期）
└── 数据表格视图（可切换）
```

#### 数据结构

```typescript
interface DownloadMetrics {
  daily: number;           // 今日下载量
  total: number;           // 累计下载量
  dailyAverage: number;    // 日均下载量
  growthRate: number;      // 增长率 (%)
  trend: DailyDataPoint[]; // 趋势数据
}

interface DailyDataPoint {
  date: string;
  downloads: number;
  uniqueUsers: number;
}
```

### 2. 活跃用户分析

#### 数据维度

| 指标 | 说明 | 统计周期 |
|------|------|---------|
| DAU | 日活跃用户数 | 每日 |
| WAU | 周活跃用户数 | 每周 |
| MAU | 月活跃用户数 | 每月 |
| 留存率 | 次日/7日/30日留存 | 每日 |
| 用户来源 | 下载渠道分布 | 实时 |

#### 可视化设计

```
活跃用户面板
├── 活跃用户卡片
│   ├── DAU / WAU / MAU 指标卡
│   ├── 环比变化趋势箭头
│   └── 历史对比迷你图
├── 留存分析
│   ├── 留存曲线图
│   ├── 留存热力图（按周）
│   └── 流失预警标识
├── 用户来源
│   ├── 渠道分布饼图
│   ├── 渠道质量对比
│   └── ROI 分析
└── 用户画像
    ├── 地域分布
    ├── 使用时段
    └── 使用频率
```

#### 数据结构

```typescript
interface UserMetrics {
  dau: number;              // 日活跃用户
  wau: number;              // 周活跃用户
  mau: number;              // 月活跃用户
  retention: RetentionData; // 留存数据
  sources: SourceData[];    // 来源分布
}

interface RetentionData {
  day1: number;   // 次日留存率
  day7: number;   // 7日留存率
  day30: number;  // 30日留存率
  curve: number[]; // 留存曲线
}

interface SourceData {
  source: string;
  users: number;
  percentage: number;
  conversionRate: number;
}
```

### 3. 评分趋势

#### 数据维度

| 指标 | 说明 | 更新频率 |
|------|------|---------|
| 综合评分 | 加权平均分（1-5分） | 实时 |
| 评分数量 | 累计评分总数 | 实时 |
| 评分分布 | 各分数段占比 | 实时 |

#### 可视化设计

```
评分分析面板
├── 评分概览
│   ├── 综合评分大数字展示
│   ├── 评分星级可视化
│   └── 评分排名（同类对比）
├── 评分分布
│   ├── 柱状图（1-5分分布）
│   ├── 百分比标注
│   └── 行业平均对比线
├── 评分趋势
│   ├── 折线图（评分变化）
│   ├── 标注关键事件（版本更新等）
│   └── 预测趋势线
└── 评价分析
    ├── 词云（评价关键词）
    ├── 情感分析（正面/中性/负面）
    └── 高频问题提取
```

#### 数据结构

```typescript
interface RatingMetrics {
  overall: number;           // 综合评分

interface RatingDistribution {
  1: number;  // 1星占比

  4: number;  // 4星占比
  5: number;  // 5星占比
}

interface RatingTrend {
  date: string;
  average: number;
  count: number;
}
```

#### 数据结构

```typescript
interface RevenueMetrics {
  today: number;           // 今日收入
  thisMonth: number;       // 本月收入

  pending: number;         // 待结算

  date: string;

  bonus: number;      // 激励奖金
  refund: number;     // 退款扣除

}

interface BillItem {
  id: string;
  date: string;
  type: 'sales' | 'bonus' | 'refund';
  amount: number;
  description: string;
  productName?: string;
}
```

### 详细分析页

- **下载分析页**：下载量详细趋势、渠道分析、地域分布
- **用户分析页**：用户画像、留存分析、行为路径
- **评分分析页**：评分详情、评价分析、竞品对比
- **收入分析页**：收入明细、账单管理、提现记录

## 数据刷新策略

### 实时数据

- 下载量：WebSocket 推送，延迟 < 5 秒
- 收入：WebSocket 推送，延迟 < 10 秒
- 活跃用户：轮询，间隔 30 秒

### 准实时数据

- 评分：轮询，间隔 1 分钟
- 用户来源：轮询，间隔 5 分钟

### 批量数据

- 历史趋势：按需加载，缓存 1 小时
- 报表导出：异步生成，完成后通知

## 数据导出

### 支持格式

- CSV：通用表格格式
- Excel：带格式的报表
- PDF：可视化报表
- JSON：API 数据格式

### 导出内容

| 报表类型 | 包含数据 | 生成时间 |
|---------|---------|---------|
| 日报 | 当日所有指标 | 实时 |
| 周报 | 本周汇总 + 趋势 | 每周一 |
| 月报 | 月度汇总 + 分析 | 每月 1 日 |
| 自定义 | 用户选择的时间范围 | 按需 |

## 告警系统

### 告警规则

| 告警类型 | 触发条件 | 通知方式 |
|---------|---------|---------|
| 收入异常 | 日收入下降 50%+ | 邮件 + 站内信 |
| 评分下降 | 周均评分下降 0.5+ | 邮件 + 站内信 |
| 流量异常 | 下载量异常波动 | 站内信 |
| 退款预警 | 退款率超过 10% | 邮件 + 站内信 |

### 告警配置

创作者可自定义告警规则：

- 告警阈值
- 通知方式
- 免打扰时段

## 权限控制

### 数据可见性

| 角色 | 可见数据 |
|------|---------|
| 创作者本人 | 所有数据 |
| 团队成员 | 授权范围内的数据 |
| 平台运营 | 聚合数据（脱敏） |

### 数据安全

- 所有数据传输加密（HTTPS）
- 敏感数据脱敏展示
- 数据导出需二次验证
- 操作日志完整记录

## API 接口

### 获取下载量数据

```http
GET /api/v1/creator/analytics/downloads
Authorization: Bearer {token}
Query: startDate, endDate, granularity
```

### 获取活跃用户数据

```http
GET /api/v1/creator/analytics/users
Authorization: Bearer {token}
Query: startDate, endDate, metrics
```

### 获取评分数据

```http
GET /api/v1/creator/analytics/ratings
Authorization: Bearer {token}
Query: productId, startDate, endDate
```

### 获取收入数据

```http
GET /api/v1/creator/analytics/revenue
Authorization: Bearer {token}
Query: startDate, endDate, type
```

## 技术实现要点

### 性能优化

- 数据预聚合：ClickHouse 预计算常见指标
- 缓存策略：Redis 缓存热点数据
- 懒加载：图表按需渲染
- 虚拟滚动：大数据列表优化

### 可扩展性

- 指标可配置化：支持自定义指标
- 图表可插拔：支持自定义图表类型
- 报表可定制：支持自定义报表模板
