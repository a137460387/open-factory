# Code of Conduct

open-factory welcomes focused, respectful collaboration. The project deals with local media tooling, so contributors are expected to protect user privacy, avoid requesting private media samples, and keep security reports private until fixes are available.

## Expected Behavior

- Be direct, constructive, and specific when discussing code, design, bugs, or reviews.
- Respect different experience levels and keep feedback about the work, not the person.
- Share only media, logs, and project files that you are allowed to publish.
- Report security issues through the private process in `SECURITY.md`.

## Unacceptable Behavior

- Harassment, threats, insults, or repeated unwanted contact.
- Publishing another person's private information, including contact details or private media.
- Posting exploit details for unresolved security issues in public threads.
- Submitting copied code, assets, logos, or wording that the project is not allowed to use.

## Enforcement

Maintainers may edit or remove comments, close issues, block participants, or reject contributions that violate these rules. Serious security or privacy issues may be handled privately to reduce harm to users.
